Rules

  1. Players are X and O.
  2. Take turns placing marks.
  3. Get 3 in a row to win.
  4. If no winner and board is full: draw.

Data Info

  • State is stored in your Flask session (no DB).
  • No login/account required.
  • Score remains until session/cookies are cleared.

About Us

This Tic‑Tac‑Toe project is a full‑stack mini application built to demonstrate how a Python backend and a modern frontend work together. The backend (Flask) manages the rules, turn logic, and scoring, while the frontend focuses on a smooth, attractive user experience with animations and effects.

What we built

  • Home page (rules + data info + about section).
  • Game page with animated moves and win highlighting.
  • API endpoints: /api/state, /api/move, /api/restart, /api/reset.

How data is handled

  • Stored per-user in a Flask session (cookie-based).
  • No database in this version, so setup is simple.
  • No personal data is required.

Future improvements

  • AI opponent (easy/medium/hard)
  • Leaderboard using a database
  • Sound effects toggle + more animations
